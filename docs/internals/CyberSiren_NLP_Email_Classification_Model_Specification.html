CYBERSIREN — NLP EMAIL CLASSIFICATION MODEL SPECIFICATION

Document: NLP-SPEC-v1.0 Date: 2026-04-10 Scope: SVC-06 NLP Analysis — Phishing / Spam / Legitimate Email Text Classifier Classification: Internal / Graduation Project
Context: This document specifies the machine learning model to be deployed inside SVC-06 (NLP Analysis Service) as defined in ARCH-SPEC. The model runs as a Python FastAPI microservice invoked by the Go aggregator (SVC-07) over HTTP. It receives email subject + body text, classifies the email into {legitimate, spam, phishing}, extracts intent labels and urgency score, and returns a content risk score (int 0–100). Inference uses ONNX Runtime on CPU. Every design decision in this document traces to one or more papers from the 15-paper research corpus in the LITERATURE/ folder, verified by full-text reading.
Target Performance: Macro MCC ≥ 0.88  |  Phishing Recall ≥ 96%  |  Legitimate FPR < 0.5%  |  Phish-as-Legit Rate < 2.0%  |  CPU Inference P95 < 150 ms  |  ONNX INT8 Model < 200 MB

2.4 Text Preprocessing (Standardization)

Applied to ALL samples before splitting. Preprocessing design follows [3]'s methodology of handling character-level obfuscation and [6]'s subject+body concatenation.

1HTML stripping: BeautifulSoup .get_text(separator=' ')[11]: phishing uses CSS tricks to inject hidden text.
2Unicode NFKC normalization[3] §3.1.2: collapses homoglyphs (Cyrillic "а" → Latin "a").

2.6 Train / Validation / Test Split

Random splitting is unacceptable. Phishing campaigns reuse templates. If the same template appears in train and test, metrics are inflated. [7] uses 5-fold CV + 80/20 holdout. [3] uses 70/15/15 stratified. [5] uses 60/20/20 stratified. [6] uses 64/16/20.
StepActionJustification
1Campaign grouping: fingerprint = SHA256(subject + "|" + first_100_chars)Groups template variants together.
2Stratified group split: GroupShuffleSplit with groups=fingerprintAll campaign variants go to the same split.
3Split ratio: 70% train / 15% validation / 15% test[3] uses 70/15/15. Matches URL model spec convention.
4Assert zero fingerprint overlap between splitsPrevents information leakage.
5Assert class ratios within 5% of target per splitStratification quality check.

Primary: distilbert-base-uncased (66M parameters)

3.3 Multi-Task Architecture

Three task-specific heads sharing a single DistilBERT backbone. [14] reviews 28 cybersecurity MTL studies and finds "majority of multi-task models outperform single-task models." The shared backbone learns richer representations when trained on related tasks simultaneously.

HeadOutputActivationLoss FunctionEvidence
Classification 3-class: {legit, spam, phish} Softmax Focal Loss (γ=2.0) with class weights [3] uses CE. Focal Loss extends CE for hard-example mining. [8]: spam class needs extra focus (F1spam=0.383 in LLM benchmark).
Intent 11-class multi-label Sigmoid (per-class) Binary Cross-Entropy [9]: co-occurring phishing codes (Urgency + Action-Specific). Multi-label via sigmoid allows simultaneous intent detection.
Urgency Scalar 0.0–1.0 Sigmoid MSE Loss [9]: Urgency is a key phishing characterization code. [3] Table 6: urgency tokens ("immediate", "failure", "minutes") drive phishing classification.

3.4 Multi-Task Loss Combination

Homoscedastic uncertainty weighting (Kendall, Gal, Cipolla, CVPR 2018). Learns σ² per task to balance loss contributions automatically. Prevents the easier classification task from dominating training.

L_total = exp(-log_var_class) × L_class + log_var_class + exp(-log_var_intent) × L_intent + log_var_intent + exp(-log_var_urgency) × L_urgency + log_var_urgency
Where log_var_* are learnable nn.Parameter(torch.zeros(1)).
Handling missing labels: Only D3/D5 have intent/urgency annotations. Loss is masked: intent/urgency loss computed only on samples with those labels. Classification loss always computed.

4.2 Adversarial Training (FGM)

Evidence: [3] Table 2 — proposed framework (FGM + char-noise) achieves 93.2% accuracy at 20% char-level noise vs 54.5% for baseline DistilBERT. A 38.7 percentage-point improvement. Cross-domain on CEAS_08: 95.7% vs 67.6% baseline. [11] independently confirms: without adversarial training, DistilBERT drops to 85% precision at 20% noise.

[12] adds that INT8 quantization itself provides +18.68% adversarial robustness improvement — complementary to FGM training.
1
Character-level noise augmentation — 30% of training samples, ~10% character corruption per selected sample. Operations: insertion, deletion, substitution (biased toward visually similar chars: o→0, l→1, a→@). Applied to raw text before tokenization. [3] §3.2.2.
2
Clean forward pass — Compute Lclean using original embeddings. Class-weighted Focal Loss.
3
Adversarial perturbation — δ = ε · ∇eLclean / ||∇eLclean||2. Perturbation magnitude ε=0.01. Detached from computational graph (no second-order gradients). [3] Eq. 1.
4
Adversarial forward pass — eadv = e + δ. Compute Ladv.
5
Joint optimization — Ltotal = Lclean + λ·Ladv, where λ=0.5. [3] Eq. 2: "equal weighting to both loss components."

5.4 Calibration Analysis

  1. Plot reliability diagrams (calibration curves) for each class.
  2. Compute ECE (binned, 15 bins).
  3. If ECE > 0.05 → apply temperature scaling on validation set.
  4. Re-plot calibration curves after temperature scaling.
  5. Report ECE before and after.
Threshold optimization: Sweep phishing threshold from 0.1 to 0.9. For each: if P(phishing) > threshold → classify as phishing; else argmax of remaining classes. Find threshold giving Phishing Recall ≥ 0.96 with minimum FPR. Store optimal threshold in config.json.

Following [3]'s systematic comparison of LIME, SHAP, and IG on adversarially trained DistilBERT, with natural-language explanations via Flan-T5.

MethodImplementationParametersEvidence

7.2 Explanation Stability

[3] §4.2: Under character-level perturbations, the adversarially-trained framework maintains "comparatively stable attribution patterns." The baseline DistilBERT shows "substantial reordering of important tokens, polarity reversals, and attribution concentration on fragmented subword units (e.g., ac)." The proposed framework "encourages reliance on more generalizable contextual representations rather than brittle surface forms." This validates our FGM training — it produces not only better classification but also more stable explanations.

7.3 Production Explanation Output

Token-level LIME attributions feed into the API response explanation field. Top-10 tokens with attribution scores enable downstream security analysts to understand classification decisions. Following [3] Table 6's example format.

9. Design Justifications

Every major decision traced to specific paper evidence.

DecisionRationalePaper Evidence
DistilBERT over ModernBERT Most extensively benchmarked for phishing email across 5 papers. 66M vs 149M params. Known failure modes. ModernBERT newer with less email-specific validation. [1]: 98.9% email. [3]: adversarial robustness. [5]: feature extraction. [11]: enhanced variant. [6]: XAI pipeline.
DistilBERT over ALBERT DistilBERT has more adversarial training research ([3][11]). ALBERT 97.54% [15] vs DistilBERT 98.9% [1]. DistilBERT wider community support. [1]: DistilBERT 98.9%. [15]: ALBERT 97.54%. [3]: FGM validated on DistilBERT.
FGM adversarial training 38.7pp improvement under 20% noise. Cross-domain CEAS_08 28.1pp improvement. Single-step — minimal training overhead (~2x). [3] Table 2: 93.2% vs 54.5% at 20% noise. Table 7: 95.7% vs 67.6% cross-domain. [11]: confirms degradation without FGM.
Char-level noise augmentation [3] shows char-noise alone achieves 91.6% at 20% noise. Combined with FGM: 93.2%. Simulates real phishing obfuscation (o→0, l→1). [3] Table 2: char-noise 91.6% vs FGM-only 60.5% at 20% noise.
Focal Loss over plain CE Focal Loss down-weights easy examples, focuses on spam↔phishing boundary. Standard γ=2.0. [8]: spam F1=0.383 shows spam boundary is hardest. Focal Loss (Lin, ICCV 2017) is strict CE generalization.
Multi-task architecture Classification + intent + urgency on shared backbone. Additional params negligible vs 66M backbone. [14]: "majority of multi-task models outperform single-task" in 28-study SLR. [9]: intent taxonomy validated at κ=0.93.
Homoscedastic uncertainty weighting Automatically learns optimal loss weights. Prevents classification from dominating. Manual weights fragile. Kendall, Gal, Cipolla (CVPR 2018). Standard MTL methodology.
Campaign-aware splitting Random splitting leaks templates across splits, inflating metrics. GroupShuffleSplit enforces campaign separation. [7]: multi-source merging creates template duplicates. [3]: stratified splitting used.
3-class (legit/spam/phish) Binary conflates spam and phishing which have different risk profiles and require different handling. [8]: 3-class is correct formulation. F1spam=0.383 shows spam needs dedicated modeling.
ONNX INT8 quantization ~48% size reduction. ~0.98% accuracy cost. Adversarial robustness improves +18.68%. [12]: DistilBERT 255→132 MB. [13]: INT8 "minimal loss" on fraud detection, <500ms mobile.
MCC as primary metric Accounts for all confusion matrix quadrants. F1 ignores TN. Accuracy treats all errors equally. Consistent with URL model spec. [4]: MCC 0.998. URL-SPEC §6.1 uses MCC. [1][15]: report accuracy + F1 + AUC but not MCC — we add it.
Head-Tail truncation (128+384) Hook/lure in opening (subject + first paragraph). CTA/link/obfuscation in footer. Pure head-truncation misses footer. [3] Table 6 Email 6: FN uses "attachment-based delivery" — footer-region signal. [6]: subject+body concatenation.
LR 2e-5 Consensus of [1] (2e-5), [3] (3e-5), [6] (2e-5), [4] (1e-5 backbone). 2e-5 is empirical median. [6]: 2e-5. [1]: 2e-5. [3]: 3e-5 on DistilBERT. Range validated across 4 papers.
LIME for production explanations Model-agnostic, focused attributions, well-aligned with phishing indicators. [3] §3.2.6: "LIME-based token attributions are selected... focused attribution scores align well with intuitive phishing indicators."

10. Limitations

#LimitationImpactMitigationEvidence
1 DistilBERT latency may exceed 200ms on some CPUs Deployment constraint Fallback to ALBERT (12M, ~80ms) [15]. Reduce max_len to 384. [2]: DistilBERT 0.85s unoptimized. [12]: ONNX INT8 reduces latency.
2 Intent/urgency training data scarce (~11K labeled) Intent/urgency heads may underperform Use as auxiliary outputs. Primary: classification head. Intent is "best effort." [14]: MTL still helps even with partial labels (masked loss).
3 English-only tokenizer on multilingual input Non-English emails tokenized poorly Language detection + preprocessing. Fallback to multilingual DistilBERT (134M). [2] §5.4.1: 118 FN were non-English. [5]: English-only limitation noted.
4 512 token limit loses long email content Long marketing/phishing emails truncated Head-Tail truncation. ModernBERT (8K context) as stretch goal. [4]: ModernBERT 8K context. [3]: 256 tokens sufficient for most emails.
5 Adversarial evasion evolves continuously New obfuscation bypasses trained defenses FGM + char-noise training. Unicode normalization. Monthly retraining. [3] §5.1: "do not fully capture optimized adversarial attacks, semantic paraphrasing."
6 40/40/20 balance ≠ production distribution (~85/5/10) Threshold calibration required Optimize threshold on production-representative validation set. [4]: class imbalance >99% legitimate in real traffic vs balanced benchmarks.
7 Subtle spear-phishing with "neutral tone" evades detection FN on sophisticated attacks Multi-modal analysis (URL + header + NLP combined in SVC-07 aggregator). [3] Table 6 Email 6: FN at 0.71 confidence. [2] §5.4.1: 25 subtle landing pages missed.
